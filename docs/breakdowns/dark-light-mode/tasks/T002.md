# T002: Enable Tailwind Class-Based Dark Mode

## State: `pending`

## Action
Update `app/globals.css` to use class-based dark mode instead of media-query.

## Business Summary
Allows manual theme override. Without this, users can only follow their OS preferenceâ€”no Light/Dark toggle would work.

## Logic
- Remove `@media (prefers-color-scheme: dark)` block
- Add `.dark` selector for dark CSS variables
- Tailwind v4 uses `@custom-variant` or relies on the `dark:` prefix working with `.dark` class on `<html>`

## Technical Logic
Tailwind v4 class-based dark mode:
```css
:root {
  --background: #ffffff;
  --foreground: #171717;
}

.dark {
  --background: #0a0a0a;
  --foreground: #ededed;
}
```

The `dark:*` utility classes automatically apply when `.dark` is present on an ancestor (default behavior in Tailwind v4).

## Testing
Manual: Add `class="dark"` to `<html>` in devtools and verify colors change.
