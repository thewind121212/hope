# T008: Audit Dark Mode Styles

## State: `pending`

## Action
Review all UI components and screens for proper dark mode support.

## Business Summary
Ensures a polished user experience in both themes. No component should become unreadable or visually broken in dark mode.

## Logic
Verify the following have appropriate `dark:*` classes:
- `components/ui/Button.tsx` — all variants
- `components/ui/Input.tsx`
- `components/ui/Textarea.tsx`
- `components/ui/Select.tsx`
- `components/ui/Modal.tsx`
- `components/ui/BottomSheet.tsx`
- `components/ui/Card.tsx`
- `components/ui/Badge.tsx`
- `components/ui/EmptyState.tsx`
- `components/ui/ConfirmDialog.tsx`
- `components/ui/DropdownMenu.tsx`
- `components/ui/KeyboardShortcutsHelp.tsx`
- `components/bookmarks/*` — all bookmark-related components
- `app/layout.tsx` — body, header
- `app/page.tsx` — main content area

## Technical Logic
Check for:
- Background colors: `bg-*` should have `dark:bg-*`
- Text colors: `text-*` should have `dark:text-*`
- Border colors: `border-*` should have `dark:border-*`
- Focus rings: `ring-*` should have `dark:ring-*`
- Sufficient contrast ratios in both modes

## Testing
Manual: Toggle between light/dark and visually inspect every screen and component.

## Patterns
Reference existing `dark:*` usage in `Button.tsx` and `layout.tsx`.
